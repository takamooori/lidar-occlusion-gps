# LiDAR遮蔽率 × GPS信頼度推定 プロジェクト

**最終更新: 2026-06-03（中間発表後）**

---

## 📌 1. プロジェクトの全体像

### 目的
LiDARで得た**上半球の遮蔽率**を客観的なGPS信頼度指標とし、それをEKFの観測ノイズ共分散 **R 行列の動的更新パラメータ**として使うことで、屋外自律移動ロボットの自己位置推定精度を環境に応じて適応的に向上させる。

### 新規性
> **「LiDARで得た幾何学的遮蔽率を、EKFのR行列の動的更新指標として直接使う」**

カメラベースのSky View Factor研究は「環境を見る」が、それを測位値にどう作用させるかは曖昧。本研究は LiDAR（3D・距離情報あり）で得た指標を **数式的にRに紐付け**、環境情報を測位信頼度として正しく取り込む。

### 現在のフェーズ
中間発表後、提案手法の方向性を「手法切替（Mode A）」から **「R行列動的制御」に再転換**。
- 理由①：完全切替だと座標系不一致でオドメトリが飛ぶ
- 理由②：評価者から「Rの動的更新こそ新規性」との指摘

## 🔬 2. 提案手法の核心

```
状態予測: IMU + Wheel Odometry で連続予測
観測1 (GPS):    R_GPS(occ)  = R0 × exp(α · occ)        ← 遮蔽率↑ → R↑ → 信頼度↓
観測2 (SLAM):   R_SLAM(occ) = R0 × exp(β · (1−occ))    ← 開空     → R↑ → 信頼度↓
```

**この設計の利点**：
- 共分散が連続変化 → 切替ジャンプなし
- 両観測が常に活きる（重みが0になるだけで完全停止しない）
- オドメトリは飛ばない（予測ステップは常時稼働）

| 指標 | 値 |

| Pearson r | **+0.164 (p=0.075)** |

| 決定係数 R² | **0.027** |
| 軌跡誤差 mean / max | 0.61m / 1.55m |
| 遮蔽率 mean / range | 0.28 / 0.15-0.65 |

**評価**：統計的有意性に届かず。原因は ① 遮蔽率レンジが狭い、② 上半球平均で情報量不足、③ **遮蔽率の妥当性検証がまだない**こと（最重要弱点）。

## 🎯 4. 遮蔽率の仕組み

![遮蔽率の算出メカニズム](occlusion_mechanism.png)

- フィボナッチ格子で上半球に **N=1000本のレイ** を均等分散
- 各レイが障害物に当たれば「遮蔽」、当たらなければ「開空」
- 仰角15°以下は **GPSのマスク角** に対応してカウント外
- 有効距離: **1.5m 〜 30m**

```
遮蔽率 occ = ヒットレイ数 / 1000   ∈ [0, 1]
```

## ⚠️ 10. 注意点・教訓

- **`combine_glim` 出力bagは使わない**（stamp破損）。代わりに元bag UM982 + dumpの `T_world_lidar` をオフラインアライメント
- **論文表記**：GLIMの軌跡は "**reference trajectory**"（×ground truth）。事前地図使用の制限を明示
- **`points_compact.bin` の罠**：ワールド座標累積点群であり、フレームローカルではない。フレームごとの生スキャンが必要な場合は別途読み込み
- **中庭データの限界**：遮蔽率レンジが狭い (0.15〜0.65)。高遮蔽（>0.7）・低遮蔽（<0.1）環境での追加収集が必須
